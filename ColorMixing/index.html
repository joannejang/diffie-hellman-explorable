Diffie-Hellman Key Exchange

An Explorable Explanation by Joanne Jang, Clay Jones, Jessica Ouyang

Introduction

This lesson hopes to explain the process of a Diffie Hellman key exchange. For starters, a key exchange is a method of transporting a secret code from one individual to another, so that both individuals have the same secret code. The issue is that we cannot send a randomly generated code because someone could intercept it.

For example, let’s imagine that you open up a new bank account online and your bank wants to send you your account number over the internet. Assuming that you understand the basics of encryption, you and your bank must have a shared key that the bank can use to encrypt the account number into a message and you can use to decrypt the account number from that message. The key exchange process is used to give you and your bank that shared key that no one else knows.

One-way Functions

Mixing paint is easy.

An important concept to understand when learning about the Diffie-Hellman key exchange is the nature of one-way functions. A one-way function is a mathematical function that is significantly easier to compute in one direction than in the opposite direction. One example of a one-way function is mixing colors. As you can see, we can use two colors to create a third color simply by mixing the two.
